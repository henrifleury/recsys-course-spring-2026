# A/B Experiment Report: LightGBM LambdaRank vs SasRec-I2I

## Abstract

Цель эксперимента — улучшить метрику `mean_session_time` сервиса Botify по сравнению с бейзлайном SasRec-I2I. Предлагалось использовать LightGBM LambdaRank, обученный предсказывать время прослушивания трека. Модель использовала коллаборативные эмбеддинги (TruncatedSVD), статистики треков и пользователей, а также контекстные фичи. Эксперимент показал статистически значимое ухудшение всех метрик. Потратил кучу времени на пересмотр лекций, не смог сходу завести HSTU начал метаться и навайбкодил эту шляпу. Практически первая попытка без оптимизаций. Штраф за просрочку и за недобор баллов есть, не знаю что хуже, но времени доделывать тоже нет. 

## Детали реализации

### Данные
- 10,000 пользователей, 16,198 треков, ~3.78M событий
- Каждый пользователь — одна непрерывная сессия в среднем из 378 треков
- Train/val: разбиение по времени (80/20), метрика валидации — `mean_session_time@10`

### Признаки
- **Трековые**: avg/median/std `time`, частота, recency, popularity
- **Пользовательские**: avg/median/total `time`, количество уникальных треков
- **Коллаборативные**: TruncatedSVD(64) на user-track матрице, косинусное сходство
- **Контекстные**: час дня (sin/cos), среднее время предыдущего трека, позиция в истории

### Модель
LightGBM LambdaRank с `label_gain = time × 100`. Группировка по пользователю. На валидации показала lift +14.6% над popularity baseline.

### Инференс
Для каждого пользователя предзагружен список из 200 треков, отсортированных по предсказанию модели. Рекомендер последовательно выдаёт треки из списка, пропуская уже прослушанный `prev_track`.

## Анализ причин

1. **Статический список рекомендаций**: модель выдаёт фиксированный список из 200 треков и последовательно их проходит. В отличие от SasRec-I2I, который для каждого `prev_track` подбирает релевантные треки динамически.

2. **Отсутствие персонализации в реальном времени**: при инференсе не используются контекстные фичи (час дня, позиция в сессии, prev_track), которые были важны при обучении. Модель видела их на train, но на инференсе они зафиксированы.


## Выводы

LightGBM LambdaRank с статическими рекомендациями не смог превзойти динамический SasRec-I2I. Для улучшения можно попробовать реализовать динамический инференс с учётом контекста, либо обучить модель предсказывать следующее действие с учётом всей последовательности, а не только агрегированных статистик.
